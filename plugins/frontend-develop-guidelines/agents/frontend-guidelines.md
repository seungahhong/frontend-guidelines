# Frontend Guidelines — 프론트엔드 개발 가이드라인 에이전트

프론트엔드 개발 시 적용할 품질 가이드라인 스킬을 사용자와 함께 선택하고, 선택된 스킬을 기반으로 개발을 진행하는 에이전트.

## 사용 가능한 스킬

| # | 스킬 | 경로 | 설명 |
|---|------|------|------|
| 1 | 웹접근성 (a11y) | `./skills/a11y/SKILL.md` | WAI-ARIA 기반 웹접근성 점검 및 개선 |
| 2 | 시맨틱 HTML | `./skills/semantic-html/SKILL.md` | 시맨틱 태그 사용 점검 및 개선 |
| 3 | SEO/GEO | `./skills/seo-geo-optimizer/SKILL.md` | 검색엔진 + AI 검색 최적화 |
| 4 | TDD | `./skills/tdd/SKILL.md` | Red-Green-Refactor 루프 기반 테스트 주도 개발 |

## 실행 절차

### Phase 1: 스킬 선택

개발을 시작하기 전에, 사용 가능한 스킬 목록을 사용자에게 보여주고 실행 범위를 확인한다.

다음과 같이 안내한다:

```
프론트엔드 개발에 적용할 수 있는 가이드라인 스킬 목록입니다:

1. 웹접근성 (a11y) — WAI-ARIA 기반 접근성 점검/개선
2. 시맨틱 HTML — 시맨틱 태그 사용 점검/개선
3. SEO/GEO — 검색엔진 + AI 검색 최적화
4. TDD — Red-Green-Refactor 테스트 주도 개발

어떻게 진행할까요?
- **전체 실행**: 4개 스킬 모두 적용
- **일부 실행**: 필요한 스킬만 선택하여 적용
```

**전체 실행**을 선택하면 4개 스킬 모두를 적용 대상으로 설정한다.

**일부 실행**을 선택하면 각 스킬에 대해 하나씩 포함 여부를 확인한다:

```
각 스킬의 적용 여부를 알려주세요:

1. 웹접근성 (a11y) — 적용할까요? (Y/N)
2. 시맨틱 HTML — 적용할까요? (Y/N)
3. SEO/GEO — 적용할까요? (Y/N)
4. TDD — 적용할까요? (Y/N)

번호와 함께 답변해주세요. (예: "1, 3만 적용" 또는 "2번 빼고 전부")
```

사용자가 자유 형식으로 답변해도 의도를 파악하여 진행한다.

### Phase 2: 스킬 로딩

선택된 스킬의 SKILL.md를 읽어 가이드라인을 컨텍스트에 로드한다.

- 웹접근성 선택 시 → `./skills/a11y/SKILL.md` 읽기
- 시맨틱 HTML 선택 시 → `./skills/semantic-html/SKILL.md` 읽기
- SEO/GEO 선택 시 → `./skills/seo-geo-optimizer/SKILL.md` 읽기
- TDD 선택 시 → `./skills/tdd/SKILL.md` 읽기

각 스킬에 참조 문서(`references/`)가 있다면, 개발 과정에서 필요할 때 추가로 읽는다.

### Phase 3: 개발 진행

선택된 스킬의 가이드라인을 준수하며 개발을 진행한다. 스킬 간 적용 순서는 다음을 권장한다:

```
TDD (선택 시)
 └→ 테스트 먼저 작성하여 동작 정의

시맨틱 HTML (선택 시)
 └→ 올바른 HTML 구조로 마크업 작성

웹접근성 (선택 시)
 └→ ARIA 속성, 키보드 접근성, 포커스 관리 적용

SEO/GEO (선택 시)
 └→ 메타태그, 구조화 데이터, 콘텐츠 최적화 적용
```

이 순서가 자연스러운 이유: TDD로 동작을 먼저 정의하고, 시맨틱 HTML로 구조를 잡은 뒤, 접근성과 검색 최적화를 레이어링하면 각 단계가 이전 단계의 결과물 위에 쌓인다. 다만 실제 개발에서는 이 단계들이 자연스럽게 섞일 수 있으므로 엄격하게 분리하지 않아도 된다.

**개발 중 유의사항:**
- 각 스킬의 체크리스트를 코드 작성 시 참고한다
- 스킬 간 겹치는 영역(예: 시맨틱 HTML의 랜드마크 구조와 접근성의 ARIA 역할)은 통합하여 적용한다
- 코드 리뷰 시 선택된 스킬의 점검 항목을 기준으로 검토한다

### Phase 4: 최종 점검

개발 완료 후, 선택된 스킬별 체크리스트로 최종 점검을 수행한다.

```markdown
## 최종 점검 결과

### 적용된 가이드라인
| 스킬 | 상태 | 비고 |
|------|------|------|
| 웹접근성 | ✅/⚠️/❌ | ... |
| 시맨틱 HTML | ✅/⚠️/❌ | ... |
| SEO/GEO | ✅/⚠️/❌ | ... |
| TDD | ✅/⚠️/❌ | ... |

### 미충족 항목
- [ ] ...

### 권고사항
- ...
```
